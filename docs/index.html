Stanford Fall 2016,

This repository contains 5 Android applications. These applications follow the guidlines suggested by Stanford's Fall 2016 CS193a with Marty Stepp, a guy who loves dogs, as the course instructor.

Note: All code has been writen by mhd53, without the use of any started code other than what is provided by Android Studio.

Application List:

  • Berkeley Admission Quiz App
  • To Do List
  • Mad Libs version 1.0
  • Mad Libs version 1.1
  • Animal Game

Berkeley Admission Quiz App

A dumb introductory application to android development filled with the hate that Stanford has for Berkeley. Joke aside, the application asks the user which of the two number buttons present on the screen is greatest. The app keeps track of points gained and lost during the play session by dynamically updating the TextView class.

To Do List

A robust applicaton that permits the user to save tasks within an ArrayList without there beining and data losses when the application gets terminated. The ArrayList is saved and is accesible for use during the next session. The ArrayList is hooked to an ArrayAdapter which is set to the listView for the user to observe the data that has been saved or added to the ArrayList.

Mad Libs version 1.0

A fun mobile version of the classical mad libs game which was invented in 1953 by Leonard Stern & Roger Price. This app contains 4 stories selected randomley everytime the user clicks on the Play button. The app asks the user to input words within a specific grammatical catergory. After asking all the required words to complete the story the user is presented with their mad lib story.

Mad Libs version 1.1

In this version of the Mad Libs app a few significant improvments have been made.

  • Added a Scroll View premitting the user to choose the story they want or if they want the computer to choose for them at random.
  • Added voice recognition, allowing the user to tell the words they want in their mad lib or type them.
  • Added Text to Speech, telling the user what type of word it requires.
  • Added a pop up window that is activated when the user clicks on the Choose New Story button, allow the user to select the story for the next session.
  • A beautiful user interface has been developed with a more consistent color scheme.

Animal Game

This is an animal game that askes the user to think about an animal. The app contains a database of Edges & Nodes. Each node contains a nodeID, a type, a text, a user-ID for further expansion of the database, a time_stamp, and a view_counter as set properties. The nodeID is used in each edge to reference which node points to which, and this is defined in each edge by the parentID & childID properties. The type property can take on only two different values, question & answer. Each edge also contains a unique graphID & a binary value type property with values, yes & no. The app cylces through the the nodes with the help of the graph depicted by the edges to reach the end node. This is all implemented using android's built in SQLite api by doing a set of queries based on the user's interaction with the app. After reaching the last node, depending on whether the app was able to guess correctly the animal that the user had in mind, the user will have the option of adding on to the database by inputting the animals name and a question that differentiates it from the animal the app guessed.
